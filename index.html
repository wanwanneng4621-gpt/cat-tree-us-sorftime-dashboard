客户需求与Kano模型

必备需求

  • 不晃、不倒、能固定到墙面。
  • 平台和猫窝尺寸适合成年猫/大猫。
  • 零件齐全，安装孔位清晰，说明书好懂。
  • 绳子、布料、抓柱耐磨，不快速掉毛掉绳。

表现型需求

  • 高度、层数和占地面积的平衡。
  • 多猫家庭的承重和动线。
  • 更好的包装防护和退换货体验。
  • 价格与实际尺寸、材质匹配。

惊喜型需求

  • 米色、浅灰、木纹、boho 等家居适配外观。
  • 可替换抓柱、可清洗垫子、模块化加配件。
  • 适合窗边、公寓小空间、短视频展示的场景化设计。

Listing & Operations

Listing页面与运营打法

  • 主图：白底展示完整高度、猫窝/平台/抓柱数量；避免看起来比实际更大。
  • 卖点图：重点放大加厚底板、墙面固定带、平台尺寸、抓柱材质、安装步骤。
  • 场景图：美国公寓客厅、窗边、小户型角落，多猫/大猫使用场景。
  • 标题词：cat tree, cat tree tower, cat tower for indoor cats, large cat tree, cat condo。
  • 广告：先用 exact/phrase 验证 `cat tree`、`cat tree tower`、`cat tree for large cats`；竞品ASIN投放优先打评分低、稳定性差、图片弱的Listing。
  • 转化监控：重点看主图CTR、详情页转化率、差评中的“wobbly / too small / missing parts / rope falls apart”。

Risk Control

风险提示与人工复核点

  • 数据风险：Sorftime 月销量和销售额是第三方估算，适合做相对比较，不等同于卖家后台真实订单。
  • Linkfox风险：Amazon实时前台搜索本轮受402余额限制，补余额后需要重跑前5页SERP，复核广告位和自然位。
  • 安全风险：Cat tree 客诉高频在稳定性、坠落、尺寸不适合大猫，结构测试优先级高于外观。
  • 物流风险：大尺寸款包装体积、破损和退货成本高，首批不宜盲目做超高/超重款。
  • 合规风险：不要在Listing中做未经验证的承重、医疗、训练等过度声明；图片上线前用Ruiguan做相似违规筛查。
  • 供应链复核：需要供应商提供承重测试、摇晃测试、材质/气味控制、跌落包装测试记录。
